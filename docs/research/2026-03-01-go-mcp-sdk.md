### Why This Over `mark3labs/mcp-go`

`mark3labs/mcp-go` (8.3k stars, v0.44.1) existed first and has more stars, but:

- Never reached v1.0 (still pre-release)
- GitHub's own MCP Server [migrated away from it](https://github.com/github/github-mcp-server/issues/1184)
- The official SDK has semver stability guarantees
- Same transport support, same feature set

### Transports

| Transport | Type | Use Case |
|-----------|------|----------|
| `StdioTransport` | stdio | Claude Code, CLI tools |
| `CommandTransport` | stdio (subprocess) | Client launching a server process |
| `StreamableHTTPHandler` | HTTP + streaming | Web-hosted MCP servers |
| `SSEHandler` | HTTP + SSE | Legacy web transport |
| `IOTransport` | arbitrary `io.ReadWriter` | Custom pipes, sockets |
| `InMemoryTransport` | in-process | Testing |

### Minimal Server

```go
package main

import (
    "context"
    "log"

    "github.com/modelcontextprotocol/go-sdk/mcp"
)

type GreetInput struct {
    Name string `json:"name" jsonschema:"the name of the person to greet"`
}

func main() {
    server := mcp.NewServer(
        &mcp.Implementation{Name: "icloud-mcp", Version: "v0.1.0"},
        nil,
    )

    mcp.AddTool(server, &mcp.Tool{
        Name:        "greet",
        Description: "Greet someone by name",
    }, func(ctx context.Context, req *mcp.CallToolRequest, input GreetInput) (*mcp.CallToolResult, any, error) {
        return &mcp.CallToolResult{
            Content: []mcp.Content{
                &mcp.TextContent{Text: "Hello, " + input.Name + "!"},
            },
        }, nil, nil
    })

    if err := server.Run(context.Background(), &mcp.StdioTransport{}); err != nil {
        log.Fatal(err)
    }
}
```

### Key API Patterns

- `mcp.AddTool(server, toolDef, handler)` — generic function, infers JSON Schema from input struct
- Struct tags: `json:"name"` for field name, `jsonschema:"description"` for tool parameter descriptions
- Handler signature: `func(ctx, *CallToolRequest, T) (*CallToolResult, any, error)`
- `server.Run(ctx, transport)` blocks on the transport

### Recent Breaking Changes (v1.4.0)

Both opt-out via `MCPGODEBUG` env var until v1.6.0:

- DNS rebinding protection enabled by default for localhost servers
- JSON content no longer HTML-escaped by default
- Case-sensitive JSON unmarshaling enforced (CVE-2026-27896 fix)
